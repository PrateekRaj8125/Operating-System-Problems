# Security Policy

## 📢 Reporting a Vulnerability

We take security and code reliability seriously.

If you discover a vulnerability, bug, or unexpected behavior in this repository, please follow responsible disclosure:

---

### 🚫 Do NOT

- Publicly disclose the issue before it is reviewed and resolved

---

### ✅ Do

1. Create a [GitHub Security Advisory](https://github.com/PrateekRaj8125/Operating-System-Problems/security/advisories)
    OR
2. Report via email: <prateek.raj.indt@gmail.com>

---

## 📄 What to Include in Your Report

Please include:

- Clear description of the issue
- Steps to reproduce
- Expected vs actual behavior
- Potential impact
- Suggested fix (if available)

---

## 🔔 What to Expect

- ⏱ Acknowledgment within **48 hours**
- 🔍 Investigation within **7 days**
- 🛠 Fix release if the issue is confirmed
- 🙌 Credit (optional)

---

## 🎯 Scope

This repository is educational and includes:

- Shell scripts
- CPU scheduling algorithms
- Memory management programs
- Deadlock handling implementations

Potential issues may include:

- Logical errors in algorithms
- Incorrect outputs
- Unsafe memory usage (in C programs)
- Script execution vulnerabilities

---

✅ Responsible reporting helps maintain a safe and reliable learning resource.
